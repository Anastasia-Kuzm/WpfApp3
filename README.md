# Smart Notes – WPF-приложение для управления заметками

## Как запустить
1. Скачайте проект и откройте файл `WpfApp3.sln` в Visual Studio.
2. Нажмите **«Восстановить»**, когда появится запрос о восстановлении NuGet-пакетов.
3. Нажмите **F5** или кнопку **«Запуск»** для сборки и запуска приложения.

## Что нужно для работы
- **Visual Studio 2022 или новее**
- **.NET Framework 4.7.2**
- Доступ в интернет для восстановления NuGet-пакетов при первом запуске
- SQLite

## Ответы на вопросы

1. **Где реализована логика работы с заметками и почему она вынесена отдельно?**  
   Логика работы с заметками реализована в классе `NoteRepository`, реализующем интерфейс `INoteRepository`. Это позволяет доступ к данным от остальных слоёв приложения. Модель `Note` содержит только данные, а работа с базой данных полностью инкапсулирована в репозитории это упрощает тестирование.

2. **Как обеспечивается актуальность отображаемых данных при изменении заметок?**  
   После любого действия с заметкой вызывается метод `LoadNotes()`, который повторно загружает и фильтрует список заметок из репозитория. Сортировка выполняется на уровне SQL-запроса: сначала закреплённые (`IsPinned DESC`), затем по дате изменения (`ModifiedDate DESC`).

3. **Как реализована валидация данных при создании/редактировании заметки?**  
   Валидация реализована в методе `SaveNote()` окна `EditNoteWindow`. Проверяются:
   - Наличие заголовка (не пустой, не более 100 символов)
   - Максимальная длина содержания (не более 10 000 символов)
   - Формат тегов — разрешены только буквы, цифры, пробелы, запятые, дефисы и подчёркивания

4. **Как можно оптимизировать приложение при работе с большим количеством заметок?**  
   Текущая реализация загружает все заметки в память, что может привести к замедлению UI при большом объёме данных. Для оптимизации можно:
   - Ввести **постраничную загрузку** или **виртуализацию** в `ListView`.
   - Перенести фильтрацию и поиск **непосредственно в SQL-запросы** в `NoteRepository`
